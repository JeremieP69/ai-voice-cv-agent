## Current BMAD experiment

Jérémie is currently testing the development of a complete project using the BMAD workflow and AI-native development practices.

The goal is to evaluate how far an agentic workflow can support an end-to-end product development cycle, from early discovery to full implementation.

The workflow being tested includes:
- client/project discovery
- problem framing
- project brief
- PRD creation
- functional scope
- user stories
- prioritization
- development planning
- implementation support
- iteration

## Identified workflow gap

During this experiment, Jérémie identified a gap between product specifications and UI-driven development.

More specifically, the workflow could be improved around:
- creating useful UI prototypes from discovery and PRD outputs
- using UI prototypes as a stronger input for user story development
- helping AI coding agents better understand what to build from a visual prototype
- reducing the gap between written requirements and actual front-end implementation

## Current exploration

Jérémie is exploring how to improve this workflow by designing an agent-based process that connects:
1. discovery and PRD outputs
2. UI prototype generation
3. user story refinement
4. implementation guidance
5. development feedback loops

## Goal

The goal is to structure, test and improve this workflow, then potentially submit it to the BMAD community.
